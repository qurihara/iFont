# 次段の設計方針 — 2文字課題・歌声合成・競技かるた優先 (2026-06-26)

別セッションで再開するための設計メモ。Cosense `qurihara/iFont` 末尾にも同内容あり。

## ユーザー新方針（要約）

- 音声は `say` より高品質な **歌声合成（韻律=プロソディを制御できる合成）** で作りたい。
- ただし **kikiwake の実読みデータでの実験にも興味**がある。
- **百人一首のかな2連接（bigram）が、日本語全般の bigram をどれだけカバーするか**を知りたい。
  ある程度カバーできるなら kikiwake 実データの2文字課題も価値が出る。
- ただし競技かるたは**独特の韻律**で読むので一般化は限定的。
- **最優先は競技かるた応用**。早押しクイズ等への一般化ケアに膨大な労力が要るなら、
  まず競技かるた限定で実施する（段階リリース）。

## 検討論点と立案

### 1. 音源：歌声合成を主軸、kikiwake 実読みは比較/補助
- (i) **歌声合成**（NEUTRINO / Sinsy / VOICEVOX 等、韻律制御可）: 全カバレッジ + プロソディを
  独立変数として明示的に制御できる。競技かるた特有の読みを意図的に再現可能。→ **主軸**
- (ii) **kikiwake 実読み**（MFA境界済・木本さん）: 生態妥当性最高・f_audio_karuta と同一音源。
  ただし競技かるた特有の引き伸ばし/節回しで一般化は限定的。→ **比較条件**として価値

### 2. 競技かるたのプロソディ（重要）
百人一首の読みは **「1文字目と2文字目の間で音程が上がり、その後は比較的棒読み」**。
この音程上昇（ピッチアクセント的パターン）が C1→C2 の識別手掛かりになりうる。
- 歌声合成ならこのピッチ輪郭を明示的に付与/操作できる → **プロソディを独立変数**にできる。
- 設計案: frac ゲートに加えて「1→2文字目の音程上昇あり/なし」を条件にし、
  プロソディ手掛かりの寄与を切り分ける。

### 3. 百人一首 bigram カバー率調査（先に実施したい）
- 百人一首全かな列（`kimariji.json` / 各歌の読み）から **かな bigram 分布**を作る。
- 一般日本語の bigram（コーパスベース）と比較し、被覆率を出す。
- 結果次第で kikiwake 実データ2文字課題の射程が決まる。

### 4. スコープ優先順位
- **競技かるた応用が最優先**。一般化ケアが高コストなら **競技かるた限定でまず実施**。
- 段階リリース: ①競技かるた限定2文字課題 → ②一般化（早押しクイズ等）は後段。

## kikiwake MFA 資産（this session 調査結果・conda不要で利用可）

`g19922ma/Kikiwake` リポジトリに音声+MFAアライメント両方あり：
- `sounds_kimoto/*.m4a`（木本・各歌 上句/下句）, `sounds_Inaba/*.ogg`（稲葉）— **音声は repo にある**
- `mfa_aligned/001..100.TextGrid`（words + phones tier）
- `mfa_v3/onset_extracted_v3.json`（各歌 `kamigoku_onset_ms` / `words[{text,xmin_ms,xmax_ms}]` /
  `phones[{text,xmin_ms,xmax_ms}]`）← **かな/音素の ms 境界が抽出済み**
- `mfa_v3/`（`01_prepare.sh`→`02_align.sh`→`03_extract.py`, `japanese_custom.dict`）パイプライン一式
- アライメント済みは**木本さん**（corpus=木本）。**100首中18首**は引き伸ばし読み×汎用音響モデルで
  内部かな境界が破損（歌頭 onset は健全）。救済は handoff(b) 簡易onset再検出 or 話者適応再align。
- 参照: `kurihara_analysis260528/HANDOFF_260616_mfa_onset.md`, `AUDIO_PROCESSING.md`

→ **MFA を新規に走らせる必要はない**（conda 未導入のこの環境でも、既存の onset json + repo の
音声から、かな区間を切り出せる）。歌声合成を主軸にする場合は MFA すら不要（合成側で境界が既知）。

## 統一モデルでの位置づけ（再掲）

1試行 = **「先行文脈 C1 → ターゲット C2、C2 を時間ゲート」**。
- 単音（C1=∅, 文脈なし/先頭位置）= **構築済**（`say` 72字 truncation, frac 0..100% 21段階）
- 2文字（C1≠∅）= これから（**歌声合成 主軸** + kikiwake実読み 比較）
