# 事前登録ドラフト: 乙課題における提示間隔200msの干渉判定

- 対象: 聴覚版(3音連続提示)と視覚版(3文字連続提示)の乙課題
- 作成日: 2026-07-18(F2作業。統括・PIの承認前のドラフト)
- 使い方: 本文書は聴覚版・視覚版の2件の登録に分けて使う。共通部分は同一の文面で、第3節の「課題別の設定」だけを差し替える。倫理審査の承認後、募集開始前に登録する。
- 実装: 公開リポジトリ iFont の experiment/pilot_soa_audio.html / pilot_soa_visual2.html(本番モード `?prod=1`)。登録時点のコミットで刺激とプログラムを固定する。
- 検出力計算: experiment/tools/power_sim_interference.py(乱数種 20260718)。数値の出典はすべて同スクリプトの出力 experiment/tools/power_sim_interference_results.json にある。

## 1. 背景と目的

iFontプロジェクトでは、日本語かな1音(モーラ)の聞き取りやすさと、かな1文字の見分けやすさを文字単位で測定し、認識率で対応づける変換を作ることを目指している。測定の本体(Stage 2)は、刺激を1つずつ提示し、先頭から一部だけ提示して当てる課題で行う。一方、想定する応用(競技かるたのような逐次提示)では、刺激が1モーラ0.2秒の速さで連続する。

本登録の実験(乙課題)の目的は、この連続提示において「直後に次の刺激が来ること」が1つ目の刺激の認識を悪くするかどうか(干渉の有無)を、事前に定めた数値ルールで判定することである。判定の結果は、Stage 2で得る文字ごとの認識率を連続提示の場面へ適用してよいかどうかの根拠になる。

## 2. 仮説

推定対象を次のとおり定義する。

> Δ = (頭打ち条件の1番目の正答率) − (S=200msの1番目の正答率)

ここで「頭打ち条件」は提示間隔S=450msと700msをまとめたもの、正答率は参加者と字について平均した集団平均を指す。Δが正のとき、S=200msでの成績が劣化していることを意味する。

- 主要仮説(非劣性): Δ < δ である。非劣性マージンは δ = 5ポイント(0.05)とする(根拠は第8節)。
- 対立の状態(干渉あり): Δ > 0 である(頭打ちより悪い)。

この2つを片側検定の対として使い、第7節の3区分の判定(干渉なし・干渉あり・判定保留)を行う。

## 3. デザイン

- 1試行では3つの刺激(かな)を間隔Sで連続提示する。参加者は1番目と2番目を50音表からの強制選択で回答する(3番目は回答しない)。3番目の刺激は2番目の上書き刺激として置いてある。
- 間隔Sは7水準(50, 83, 133, 200, 300, 450, 700 ms)である。1人1セッションで、練習2問(正解表示あり)と本番42問(各水準6問、正解表示なし)を行う。所要は8分から12分程度である。
- 字の割当は、全字を混ぜた列から順に配る方式で行う。このため同一セッション内で1番目の刺激に同じ字は2回出ない。字と水準の組合せは参加者ごとに無作為で、試行順も無作為である。
- 練習2問は長めの間隔(700msと450ms)から出題する。練習は解析に使わない。

課題別の設定(登録時にどちらか一方を残す):

- 聴覚版: 刺激は68音(を・ぢ・づ・ゔは同音・弁別不能のため除外)。音声はVOICEVOX 東北きりたん(id108・ノーマル)を音高B3(246.94Hz)に平坦化し、1モーラ0.2秒に整えた音源(v3.2、ぱ行の字ごとの処方を含む)である。提示はそれぞれの音の音響的開始位置を起点に行う。ヘッドホンの使用を必須とし、同意画面のチェックで自己申告してもらう(機械的な検証手段は無いことを明記する)。
- 視覚版: 刺激は72字。PC画面上に同一位置へ順次表示する。タッチ端末(スマートフォン等)は対象外とする。

## 4. 評価項目

- 主要評価項目: 1番目の刺激の正誤(correct1)。判定にはS=200msの6試行と、頭打ち(S=450msと700msの計12試行)を使う。
- 副次評価項目: 2番目の刺激の正誤(correct2)、両方正解(accBoth)、全7水準の正答率。これらはすべて記録し、記述的に報告するが、干渉判定には使わない。

2番目の正誤を主要評価項目にしない理由は次のとおりである。2番目の刺激は、1番目の回答を先に入力する間、記憶に保持しなければならない。回答順が固定されているため報告も数秒遅れる。したがってcorrect2には知覚に加えて短期記憶と報告過程の影響が混ざり、提示間隔の効果を純粋に反映しない。correct1は提示の直後に最初に報告されるため、この交絡が最も小さい。

## 5. 参加者と必要数

- Yahoo!クラウドソーシングで、日本語を母語とする成人を募集する。PC限定とする。1人1回のみ参加できる。
- 有効参加者数(除外後)の目標: 聴覚版 N=130、視覚版 N=80。
- 除外の発生(1割から2割)を見込み、聴覚150人・視覚95人の枠で募集を開始し、有効数が目標に達するまで追加募集する。目標を超えて集まった適格セッションも、除外基準に該当しない限りすべて解析する。
- 中間解析は行わない(固定Nデザイン)。判定は有効数が目標に達した時点で締めたデータに対して1回だけ行う。
- 報酬は作業時間(10分前後)とプラットフォームの相場に応じて別途定める(予算設計文書に従う)。

## 6. 除外基準

セッション(参加者)単位で、次のいずれかに該当するものを解析から除外する。判定はすべてサーバに保存されたデータ(soa_trialsシートとsoa_sessionsシート)から機械的に行える。

1. セッション完了の記録(完了コード)がsoa_sessionsに無い(途中離脱)。
2. クラウドソーシング側に提出された完了コードが、サーバに記録された完了コードと一致しない。
3. 同一のworker_idによる2回目以降のセッション(最初に完了したセッションのみ採用する)。
4. 機械的回答の疑い: 本番42試行のうち、1番目の回答(resp1)の最頻の1文字が25試行(約60%)以上を占める。または、resp1とresp2の組がすべての試行で同一である。
5. 所要時間の外れ値: soa_sessionsに記録された所要秒が300秒(5分)未満、または2400秒(40分)超。根拠として、研究者パイロットの実測は8分から12分であり、刺激提示と2回の回答という手続き上、5分未満での誠実な完了は成立しない。
6. タッチ端末での参加(記録されたenv.touchがtrue)。募集文でもPC限定を明示する。

補足: 乙課題は全試行が本データであり、キャッチ試行(注意確認用のダミー問題)は無い。強制選択のため無回答は発生しない。試行単位の除外は行わず、採用したセッションの本番42試行はすべて解析する。除外により有効数が目標を下回る場合は追加募集で補充する。

## 7. 解析と判定ルール

### 7.1 主要解析(判定に使う)

1. 参加者iごとに差得点を計算する: d_i = (頭打ち12試行の正答率) − (S=200msの6試行の正答率)。
2. Δ^ = 平均(d_i)、SE = 標準偏差(d_i, 不偏)/√N とする。自由度N−1のt分布の95%点 t を使い、片側の限界を U = Δ^ + t·SE、L = Δ^ − t·SE とする。
3. 判定(δ = 0.05):
   - U < δ ならば「干渉なし」と判定する。
   - U ≥ δ かつ L > 0 ならば「干渉あり」と判定する。
   - どちらでもなければ「判定保留」とする。
4. U < δ と L > 0 が同時に成立した場合(統計的には劣化があるが、マージン内に収まると95%で言える場合)は、「干渉なし(ただしマージン内の小さな劣化あり)」として報告する。運用上の扱いは干渉なしと同じである。
5. 有意水準は各片側検定でα=0.05とする。これは心理学における同等性・非劣性検定の慣行(両側90%信頼区間の利用に相当)に従う。

判定の帰結もあらかじめ定める。「干渉なし」ならStage 2は0.2秒刺激のまま進め、その結果を連続提示の場面へ適用する。「干渉あり」なら連続提示の運用速度の見直し(間隔を広げる)か、干渉を明示した換算の追加を検討する。「判定保留」の場合、本登録の確認的な結論は保留のまま報告し、追加データによる再検証は新たな登録として行う。

この推定方式の妥当性は次のとおりである。字は参加者内で繰り返されず、字と条件の組合せは無作為なので、d_iは集団平均差Δの不偏推定である。参加者の全体的な巧拙はd_iの中で相殺される。字のばらつきが大きくても、出題方式(参加者内で字が繰り返されない)のため差得点の分散はほとんど増えない。検出力シミュレーション(第9節)で、この手続きの誤判定率が設計どおり約5%であることを確認した。

### 7.2 補助解析(推定の精緻化。判定は7.1が優先)

- 階層ロジスティック回帰を推定する: correct1 ~ 条件(S=200 vs 頭打ち) + (1|参加者) + (1|字)。ベイズ推定(Stan系のソフトウェア)を用い、弱情報事前分布(切片 Normal(0, 1.5)、条件係数 Normal(0, 1)、ランダム効果の標準偏差 half-Normal(0, 1))を置く。ランダム効果を数値的に周辺化して集団平均の正答率差Δの事後分布を求め、P(Δ < δ) と P(Δ > 0) を報告する。P(Δ < δ) ≥ 0.95 を「干渉なし」、P(Δ > 0) ≥ 0.95 かつ非劣性不成立を「干渉あり」に対応づける。
- 7.1と7.2の判定が食い違った場合は両方を報告し、設計上の意思決定には慎重な側(干渉あり、判定保留、干渉なし の順に慎重とする)を採用する。
- 頭打ちの確認(操作チェック): S=450とS=700の対の差(参加者ごと各6試行)を7.1と同じt手続きで推定する。両側95%信頼区間が0を除外し、かつ点推定の絶対値が5ポイントを超える場合は、頭打ちの仮定に疑義があるため、頭打ちをS=700のみ(およびS=450のみ)とした感度分析を行い、主要判定と食い違えば慎重な側を採用する。

## 8. 非劣性マージン δ = 5ポイント の根拠

1. Stage 2の測定精度との関係。Stage 2では字ごとの正答率を推定して変換gの入力にする。1字1条件あたりの回答数をnとすると、正答率0.7付近の標準誤差は √(0.21/n) であり、現実的な n = 40〜60 で6.5から8ポイントになる。5ポイント未満の一様な低下は、gが入力として使う字ごとの推定値の精度より小さく、gの出力(聴覚と視覚の対応づけ)を実質的に変えない。
2. 変換gの構造。gは両モダリティの認識率の順位づけと率合わせで決まるため、全字へ一様にかかる小さな低下は字の順位を変えない。字によって異なる干渉が起きる場合はΔの平均にも現れやすく、さらに補助解析の字ランダム効果と感度分析5(字別の探索)で点検する。
3. パイロットの実測との整合。研究者1名の3セッション合算で、頭打ちは27/36 = 75.0%(Wilson 95%区間 58.9〜86.3)、S=200msは14/18 = 77.8%(同 54.8〜91.0)であり、点推定の差は −2.8ポイント(S=200のほうが良い)だった。真の差が0付近であることと矛盾しない。一方で8ポイント以上の劣化は、認識率を文字ごとに積み上げるこの応用では文字列全体の成功率へ直接効くため、見逃したくない大きさとして検出目標(第9節)に置いた。

## 9. 検出力と必要Nの根拠

方法: 生成モデルを logit P(correct1) = α + a_i + b_c − β·1[S=200] とする。a_iは参加者のランダム効果(SD 0.6)、b_cは字のランダム効果(SD 1.2)で、αとβは周辺(集団平均)正答率が狙いの値になるよう数値求解で較正した。判定手続きは7.1をそのまま適用し、1万反復のモンテカルロで判定確率を求めた(スクリプトと乱数種は冒頭に記載)。

パラメータの根拠は次のとおりである。頭打ちの正答率は、パイロット合算の75.0%に対し、クラウド参加者が研究者より不慣れであることを見込んで聴覚70%に置いた(感度分析で60%と80%を確認)。視覚はパイロット(v1.7)で誤りが濁点・半濁点の混同に集中し曲線が単調だったことから90%に置いた(感度分析で85%と95%を確認)。字と参加者のばらつきは直接推定できるデータが無いため仮値とし、字SDを0.8〜1.6、参加者SDを0.3〜1.0に振っても判定確率がほとんど動かないことを確認した(変化は2ポイント以内。字のばらつきが効かない理由は7.1の末尾に述べた設計上の性質による)。干渉の大きさに個人差(SD 0.2、確率スケールで約4ポイント)がある場合も結果は変わらなかった。実際の出題(68字の有限プールを参加者間で共有)を模したシミュレーションと、字を毎回新規に引く近似との差も1ポイント未満だった。

主要な結果(δ = 5ポイント、判定確率は%):

| 課題 | N | 干渉なし判定(真の差0) | 干渉あり判定(真の差8pt) | 干渉あり誤判定(真の差0) | 干渉なし誤判定(真の差5pt) |
|---|---|---|---|---|---|
| 聴覚 | 100 | 71.1 | 96.3 | 5.0 | 4.9 |
| 聴覚 | **130** | **81.2** | **98.9** | **4.7** | **5.0** |
| 聴覚 | 150 | 86.2 | 99.4 | 4.9 | 4.9 |
| 視覚 | 60 | 81.3 | 97.2 | 4.5 | 5.8 |
| 視覚 | **80** | **90.2** | **99.4** | **4.5** | **6.0** |
| 視覚 | 100 | 94.8 | 99.9 | 3.7 | 5.5 |

悲観的な感度設定でも、聴覚N=130は頭打ち60%のとき75.4%、視覚N=80は頭打ち85%のとき79.5%の「干渉なし」判定力を保つ(いずれも真の差0のとき)。

以上より、有効参加者数を聴覚 N=130、視覚 N=80 と定める。これは「真の差が0のとき正しく干渉なしと判定する確率」をおおむね80%以上、「真の差が8ポイントのとき干渉ありと検出する確率」を99%程度にする最小の水準である。誤って通す方向の危険(真の劣化を見逃して干渉なしと判定する確率)は、真の差5ポイントで5%、8ポイントでは0.1%未満に抑えられる。

現行の42問構成を保つ根拠も記す。水準あたりの試行数を9問(63問、13〜18分)に増やすと、同じ判定力に必要なNは約85人まで減るが、1人あたりの作業時間と報酬がほぼ比例して増えるため総費用はほとんど変わらず、セッションが望ましい上限(15分)を超える。したがって現行構成(42問、8〜12分)を保ち、参加者数で判定力を確保する。

## 10. 多重性の扱い

聴覚版と視覚版は、刺激も参加者も帰結する設計判断も別である独立した実験であり、別々に登録して各登録内で誤り率を管理する。登録間の補正は行わない。各登録内の2つの片側検定(非劣性と劣化)は3区分判定の相補的な部分であり、それぞれの誤り率を片側5%で管理する。シミュレーションにより、「真の差が0なのに干渉ありと判定する」確率と「真の差がδなのに干渉なしと判定する」確率がいずれも約5%であることを確認した(第9節の表)。

## 11. 感度分析(事前指定)

1. 頭打ちをS=700のみ、またはS=450のみとして主要解析を再実行する(7.2の操作チェックと連動)。
2. 補助解析(階層ロジスティック回帰)で判定の再現を確認する。
3. 除外基準を緩めた場合(所要時間の下限を外す、タッチ端末を含める)と厳しくした場合(最頻回答が40%以上で除外)で再解析する。
4. 本番の前半21問と後半21問に分けて、練習効果と疲労の有無を記述的に点検する。
5. 字ごとの差を部分プーリングで探索する(字により干渉が異なるかの点検。確認的な判定には使わない)。

## 付録A: 予算制約時の代替案(登録前にどちらか一方を選び、他方の記載を削除する)

予算の制約が強い場合の代替として、δ = 8ポイント、聴覚 N=60、視覚 N=40 の設計も成立する。このとき真の差0で「干渉なし」と判定する確率は聴覚86.5%、視覚94.4%である。ただし検出の目標は10ポイントの劣化に緩み(聴覚N=60で94.6%)、8ポイント差はマージン上にあるため判定が安定しない。マージンの操作上の意味(第8節)を保てるのは主案(δ=5ポイント)であり、本文の主案を推奨する。
