# 事前登録ドラフト: 乙課題における提示間隔200msの干渉判定

- 対象: 聴覚版(3音連続提示)と視覚版(3文字連続提示)の乙課題
- 作成日: 2026-07-18(F2作業。統括・PIの承認前のドラフト)
- 使い方: 本文書は聴覚版・視覚版の2件の登録に分けて使う。共通部分は同一の文面で、第3節の「課題別の設定」だけを差し替える。倫理審査の承認後、募集開始前に登録する。
- 実装: 公開リポジトリ iFont の experiment/pilot_soa_audio.html / pilot_soa_visual2.html(本番モード `?prod=1`)。登録時点のコミットで刺激とプログラムを固定する。
- 検出力計算: 二段階手続きの判定確率は experiment/tools/power_sim_twostage.py(乱数種 20260723)、生成モデルの較正と一段版の参照値は experiment/tools/power_sim_interference.py(乱数種 20260718)。数値の出典はそれぞれの出力JSONにある。
- 設計の確定: 2026-07-23 に PI が二段階の主案(第一段δ=8ポイント・保留時のみ増員してδ=5ポイント)を採用した。判断材料は temp/乙課題_論文記載比較_主案vs代替案B.docx。

## 1. 背景と目的

iFontプロジェクトでは、日本語かな1音(モーラ)の聞き取りやすさと、かな1文字の見分けやすさを文字単位で測定し、認識率で対応づける変換を作ることを目指している。測定の本体(Stage 2)は、刺激を1つずつ提示し、先頭から一部だけ提示して当てる課題で行う。一方、想定する応用(競技かるたのような逐次提示)では、刺激が1モーラ0.2秒の速さで連続する。

本登録の実験(乙課題)の目的は、この連続提示において「直後に次の刺激が来ること」が1つ目の刺激の認識を悪くするかどうか(干渉の有無)を、事前に定めた数値ルールで判定することである。判定の結果は、Stage 2で得る文字ごとの認識率を連続提示の場面へ適用してよいかどうかの根拠になる。

## 2. 仮説

推定対象を次のとおり定義する。

> Δ = (頭打ち条件の1番目の正答率) − (S=200msの1番目の正答率)

ここで「頭打ち条件」は提示間隔S=450msと700msをまとめたもの、正答率は参加者と字について平均した集団平均を指す。Δが正のとき、S=200msでの成績が劣化していることを意味する。

- 主要仮説(非劣性): Δ < δ である。非劣性マージンδは二段階で用いる。第一段は δ = 8ポイント(0.08)、第一段が保留の場合の第二段(増員後)は δ = 5ポイント(0.05)とする(根拠は第8節、手続きは第7節)。
- 対立の状態(干渉あり): Δ > 0 である(頭打ちより悪い)。

この2つを片側検定の対として使い、第7節の二段階・3区分の判定(干渉なし・干渉あり・判定保留)を行う。

## 3. デザイン

- 1試行では3つの刺激(かな)を間隔Sで連続提示する。参加者は1番目と2番目を50音表からの強制選択で回答する(3番目は回答しない)。3番目の刺激は2番目の上書き刺激として置いてある。
- 間隔Sは7水準(50, 83, 133, 200, 300, 450, 700 ms)である。1人1セッションで、練習2問(正解表示あり)と本番42問(各水準6問、正解表示なし)を行う。所要は8分から12分程度である。
- 字の割当は、全字を混ぜた列から順に配る方式で行う。このため同一セッション内で1番目の刺激に同じ字は2回出ない。字と水準の組合せは参加者ごとに無作為で、試行順も無作為である。
- 練習2問は長めの間隔(700msと450ms)から出題する。練習は解析に使わない。

課題別の設定(登録時にどちらか一方を残す):

- 聴覚版: 刺激は68音(を・ぢ・づ・ゔは同音・弁別不能のため除外)。音声はVOICEVOX 東北きりたん(id108・ノーマル)を音高B3(246.94Hz)に平坦化し、1モーラ0.2秒に整えた音源(v3.2、ぱ行の字ごとの処方を含む)である。提示はそれぞれの音の音響的開始位置を起点に行う。ヘッドホンの使用を必須とし、同意画面のチェックで自己申告してもらう(機械的な検証手段は無いことを明記する)。
- 視覚版: 刺激は72字。PC画面上に同一位置へ順次表示する。タッチ端末(スマートフォン等)は対象外とする。

## 4. 評価項目

- 主要評価項目: 1番目の刺激の正誤(correct1)。判定にはS=200msの6試行と、頭打ち(S=450msと700msの計12試行)を使う。
- 副次評価項目: 2番目の刺激の正誤(correct2)、両方正解(accBoth)、全7水準の正答率。これらはすべて記録し、記述的に報告するが、干渉判定には使わない。

2番目の正誤を主要評価項目にしない理由は次のとおりである。2番目の刺激は、1番目の回答を先に入力する間、記憶に保持しなければならない。回答順が固定されているため報告も数秒遅れる。したがってcorrect2には知覚に加えて短期記憶と報告過程の影響が混ざり、提示間隔の効果を純粋に反映しない。correct1は提示の直後に最初に報告されるため、この交絡が最も小さい。

## 5. 参加者と必要数

- Yahoo!クラウドソーシングで、日本語を母語とする成人を募集する。PC限定とする。1人1回のみ参加できる。
- 有効参加者数(除外後)の目標は二段階で定める。第一段は聴覚 N=60、視覚 N=40(除外1〜2割を見込み、募集枠は聴覚71人・視覚48人)。第一段の判定が保留となった場合に限り、第二段として聴覚 N=130、視覚 N=80まで追加募集する(募集枠150人・95人)。
- 目標を超えて集まった適格セッションも、除外基準に該当しない限りすべて解析する。
- 各段の内部では中間解析を行わない。判定は各段の有効数が目標に達した時点で締めたデータに対して行う(第一段で1回、保留の場合のみ第二段で1回。誤り率の管理は第10節)。
- 報酬は作業時間(10分前後)とプラットフォームの相場に応じて別途定める(予算設計文書に従う)。

## 6. 除外基準

セッション(参加者)単位で、次のいずれかに該当するものを解析から除外する。判定はすべてサーバに保存されたデータ(soa_trialsシートとsoa_sessionsシート)から機械的に行える。

1. セッション完了の記録(完了コード)がsoa_sessionsに無い(途中離脱)。
2. クラウドソーシング側に提出された完了コードが、サーバに記録された完了コードと一致しない。
3. 同一のworker_idによる2回目以降のセッション(最初に完了したセッションのみ採用する)。
4. 機械的回答の疑い: 本番42試行のうち、1番目の回答(resp1)の最頻の1文字が25試行(約60%)以上を占める。または、resp1とresp2の組がすべての試行で同一である。
5. 所要時間の外れ値: soa_sessionsに記録された所要秒が300秒(5分)未満、または2400秒(40分)超。根拠として、研究者パイロットの実測は8分から12分であり、刺激提示と2回の回答という手続き上、5分未満での誠実な完了は成立しない。
6. タッチ端末での参加(記録されたenv.touchがtrue)。募集文でもPC限定を明示する。

補足: 乙課題は全試行が本データであり、キャッチ試行(注意確認用のダミー問題)は無い。強制選択のため無回答は発生しない。試行単位の除外は行わず、採用したセッションの本番42試行はすべて解析する。除外により有効数が目標を下回る場合は追加募集で補充する。

## 7. 解析と判定ルール

### 7.1 主要解析(判定に使う)

1. 参加者iごとに差得点を計算する: d_i = (頭打ち12試行の正答率) − (S=200msの6試行の正答率)。
2. Δ^ = 平均(d_i)、SE = 標準偏差(d_i, 不偏)/√N とする。自由度N−1のt分布の97.5%点 t を使い、片側の限界を U = Δ^ + t·SE、L = Δ^ − t·SE とする(各段とも片側α=0.025。配分の根拠は第10節)。
3. 第一段の判定(聴覚N=60・視覚N=40、δ1 = 0.08):
   - U < δ1 ならば「干渉なし」と判定して終了する。
   - U ≥ δ1 かつ L > 0 ならば「干渉あり」と判定して終了する。
   - どちらでもなければ「保留」とし、第二段に進む。
4. 第二段の判定(聴覚N=130・視覚N=80まで増員、δ2 = 0.05): 第一段の参加者と追加参加者の全データをプールし、手続き1〜2を繰り返す。U < δ2 で「干渉なし」、U ≥ δ2 かつ L > 0 で「干渉あり」、どちらでもなければ最終判定を「保留」とする。
5. いずれの段でも、U < δ と L > 0 が同時に成立した場合(統計的には劣化があるが、当該段のマージン内に収まると言える場合)は、「干渉なし(ただしマージン内の小さな劣化あり)」として報告する。運用上の扱いは干渉なしと同じである。

判定の帰結もあらかじめ定める。「干渉なし」ならStage 2は0.2秒刺激のまま進め、その結果を連続提示の場面へ適用する(第一段で確定した場合の主張は「劣化は8ポイント未満」、第二段で確定した場合は「5ポイント未満」である)。「干渉あり」なら連続提示の運用速度の見直し(間隔を広げる)か、干渉を明示した換算の追加を検討する。第二段でも「保留」の場合、本登録の確認的な結論は保留のまま報告し、追加データによる再検証は新たな登録として行う。

この推定方式の妥当性は次のとおりである。字は参加者内で繰り返されず、字と条件の組合せは無作為なので、d_iは集団平均差Δの不偏推定である。参加者の全体的な巧拙はd_iの中で相殺される。字のばらつきが大きくても、出題方式(参加者内で字が繰り返されない)のため差得点の分散はほとんど増えない。検出力シミュレーション(第9節)で、この手続きの誤判定率が設計どおり約5%であることを確認した。

### 7.2 補助解析(推定の精緻化。判定は7.1が優先)

- 階層ロジスティック回帰を推定する: correct1 ~ 条件(S=200 vs 頭打ち) + (1|参加者) + (1|字)。ベイズ推定(Stan系のソフトウェア)を用い、弱情報事前分布(切片 Normal(0, 1.5)、条件係数 Normal(0, 1)、ランダム効果の標準偏差 half-Normal(0, 1))を置く。ランダム効果を数値的に周辺化して集団平均の正答率差Δの事後分布を求め、P(Δ < δ) と P(Δ > 0) を報告する(δは当該段のマージン)。P(Δ < δ) ≥ 0.975 を「干渉なし」、P(Δ > 0) ≥ 0.975 かつ非劣性不成立を「干渉あり」に対応づける(主要解析の片側α=0.025に合わせる)。
- 7.1と7.2の判定が食い違った場合は両方を報告し、設計上の意思決定には慎重な側(干渉あり、判定保留、干渉なし の順に慎重とする)を採用する。
- 頭打ちの確認(操作チェック): S=450とS=700の対の差(参加者ごと各6試行)を7.1と同じt手続きで推定する。両側95%信頼区間が0を除外し、かつ点推定の絶対値が5ポイントを超える場合は、頭打ちの仮定に疑義があるため、頭打ちをS=700のみ(およびS=450のみ)とした感度分析を行い、主要判定と食い違えば慎重な側を採用する。

## 8. 非劣性マージン(第一段δ=8ポイント・第二段δ=5ポイント)の根拠

非劣性マージンには分野を横断する既定値が存在しないため、本研究の構造から導出する。第一段のδ=8ポイントは、これを超える一様な低下が、認識率を文字ごとに積み上げる本応用で文字列全体の認識に実質的に影響しうる大きさとして定めた(下記2の順位不変性より、これ未満の一様な差は変換gの出力を変えない)。第二段のδ=5ポイントの根拠は以下の1〜3である。

1. Stage 2の測定精度との関係。Stage 2では字ごとの正答率を推定して変換gの入力にする。1字1条件あたりの回答数をnとすると、正答率0.7付近の標準誤差は √(0.21/n) であり、現実的な n = 40〜60 で6.5から8ポイントになる。5ポイント未満の一様な低下は、gが入力として使う字ごとの推定値の精度より小さく、gの出力(聴覚と視覚の対応づけ)を実質的に変えない。
2. 変換gの構造。gは両モダリティの認識率の順位づけと率合わせで決まるため、全字へ一様にかかる小さな低下は字の順位を変えない。字によって異なる干渉が起きる場合はΔの平均にも現れやすく、さらに補助解析の字ランダム効果と感度分析5(字別の探索)で点検する。
3. パイロットの実測との整合。研究者2名で計4セッションを実施した。研究者A(PI)の3セッション合算は、頭打ち27/36 = 75.0%(Wilson 95%区間 58.9〜86.3)、S=200msは14/18 = 77.8%(同 54.8〜91.0)で、差得点は −2.8ポイント(S=200のほうが良い)。研究者B(共同研究者)の1セッションは、頭打ち7/12 = 58.3%、S=200msは3/6 = 50.0%で、差得点は +8.3ポイント(ただしS=200が6試行のみのため単独では雑音の範囲。Wilson 95%区間 18.8〜81.2)。2名の合算では頭打ち34/48 = 70.8%、S=200msは17/24 = 70.8%で、差は 0.0ポイントだった。真の差が0付近であることと矛盾しない。一方で8ポイント以上の劣化は、認識率を文字ごとに積み上げるこの応用では文字列全体の成功率へ直接効くため、見逃したくない大きさとして検出目標(第9節)に置いた。

## 9. 検出力と必要Nの根拠

方法: 生成モデルを logit P(correct1) = α + a_i + b_c − β·1[S=200] とする。a_iは参加者のランダム効果(SD 0.6)、b_cは字のランダム効果(SD 1.2)で、αとβは周辺(集団平均)正答率が狙いの値になるよう数値求解で較正した。判定手続きは7.1の二段階(第一段N=60/40・δ=8pt、保留時のみ増員してプールしδ=5pt、各段片側2.5%)を1レプリケーション内で忠実に模し、1万反復のモンテカルロで判定確率を求めた(スクリプトと乱数種は冒頭に記載)。

パラメータの根拠は次のとおりである。頭打ちの正答率は、研究者2名のパイロット合算の70.8%(研究者Aは75.0%、研究者Bは58.3%)を踏まえ、聴覚70%に置いた(感度分析で60%と80%を確認)。研究者Bの58.3%は、感度分析の60%設定が仮定上の悲観でなく実在する参加者像であることを示す。視覚は研究者Bの実測で頭打ち12/12 = 100%・S=200msも6/6 = 100%(誤りはS=83ms以下の超短時間提示に限られた)であり、90%の設定はこれに対して保守的である(感度分析で85%と95%を確認)。字と参加者のばらつきは直接推定できるデータが無いため仮値とし、字SDを0.8〜1.6、参加者SDを0.3〜1.0に振っても判定確率がほとんど動かないことを確認した(変化は2ポイント以内。字のばらつきが効かない理由は7.1の末尾に述べた設計上の性質による)。干渉の大きさに個人差(SD 0.2、確率スケールで約4ポイント)がある場合も結果は変わらなかった。実際の出題(68字の有限プールを参加者間で共有)を模したシミュレーションと、字を毎回新規に引く近似との差も1ポイント未満だった。

主要な結果(二段階手続き、判定確率は%):

| 課題 | 頭打ち想定 | 第一段で「なし」確定(真の差0) | 最終「なし」(真の差0) | 最終「保留」(真の差0) | 最終「あり」(真の差8pt) |
|---|---|---|---|---|---|
| 聴覚(N=60→130) | 70% | 77.6 | **84.5** | 11.7 | **96.1** |
| 視覚(N=40→80) | 90% | 89.3 | **92.7** | 4.5 | **96.3** |

悲観的な感度設定でも判定力は保たれる: 聴覚は頭打ち65%で最終「なし」82.0%、60%でも80.7%。視覚は頭打ち85%で85.5%(いずれも真の差0のとき。全セルは付録A)。

誤って通す方向の危険は次のとおり管理される。真の差が8ポイントのとき第一段で「なし(劣化8pt未満)」と誤る確率は聴覚2.3〜2.6%、視覚2.9%(名目片側2.5%。視覚のわずかな超過は小標本の離散性によるもので、シミュレーション実測値として正直に記載する)。真の差が5ポイントのとき第二段経由で「なし(劣化5pt未満)」と誤る確率は全セルで0.6%以下である。

現行の42問構成を保つ根拠も記す。水準あたりの試行数を9問(63問、13〜18分)に増やすと、同じ判定力に必要なNは約85人まで減るが、1人あたりの作業時間と報酬がほぼ比例して増えるため総費用はほとんど変わらず、セッションが望ましい上限(15分)を超える。したがって現行構成(42問、8〜12分)を保ち、参加者数で判定力を確保する。

## 10. 多重性の扱い

聴覚版と視覚版は、刺激も参加者も帰結する設計判断も別である独立した実験であり、別々に登録して各登録内で誤り率を管理する。登録間の補正は行わない。

各登録内の多重性は2種類ある。(1) 二段階の判定: 「干渉なし」を主張する機会が最大2回あるため、Bonferroni配分により各段の検定を片側α=0.025とし、手続き全体で誤って「干渉なし」を主張する確率を5%以内に保つ(等分配分。第9節のシミュレーションで、各段の誤り率が名目どおりに制御されることを実測で確認した)。(2) 3区分判定を構成する2つの片側検定(非劣性と劣化)は相補的な主張であり、それぞれを同じ片側α=0.025で管理する。

## 11. 感度分析(事前指定)

1. 頭打ちをS=700のみ、またはS=450のみとして主要解析を再実行する(7.2の操作チェックと連動)。
2. 補助解析(階層ロジスティック回帰)で判定の再現を確認する。
3. 除外基準を緩めた場合(所要時間の下限を外す、タッチ端末を含める)と厳しくした場合(最頻回答が40%以上で除外)で再解析する。
4. 本番の前半21問と後半21問に分けて、練習効果と疲労の有無を記述的に点検する。
5. 字ごとの差を部分プーリングで探索する(字により干渉が異なるかの点検。確認的な判定には使わない)。
6. 視覚版について、記録した実測タイミング(actual_soa1)が名目から25ms(60Hzの1.5フレーム)を超えて逸脱した試行を除外して再解析する(聴覚版はWeb Audioのサンプル精度スケジュールのため対象外)。
